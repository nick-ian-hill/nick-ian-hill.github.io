Agent Experience (AX)

Designing intuitive, AI-driven workflows for complex enterprise systems.

Enterprise Content Management Systems (CMS) are powerful platforms capable of managing vast networks of content. However, this power inherently introduces complexity, creating distinct friction points for different types of users. Agent Experience (AX) shifts the cognitive load from operating the system to expressing intent.

Workflow Enhancements

  • The Occasional User: For users who only interact sporadically, the steep learning curve is a barrier. Remembering the precise steps to update a specific page can be daunting.
  • The Power User: Experienced editors often get bogged down by tedious, repetitive workflows—such as manually duplicating a suite of pages or performing repetitive content updates.
  • The Organizational Bottleneck: Organizations burn valuable editorial time on mechanical tasks like tagging taxonomy, generating image alt-text, or managing translation drops.
  • The Reporting Constraint: Extracting comprehensive data from a traditional REST API requires a massive volume of individual calls. While automating this collation via AI remains technically inefficient, the high business value of the resulting analysis makes the trade-off worthwhile.

The Human Intention

Instead of clicking through complex BluePrint hierarchies or cross-referencing schemas, the user's intent should be articulated simply in natural language:

"Help me update this campaign page, or better yet, just do it for me."

"Create draft copies of these five pages and swap out the header images."

"Audit this folder and generate alt-text for all images currently missing descriptions."

The Agentic Architecture

To translate these simple intentions into reliable system actions, I architected a dedicated Model Context Protocol (MCP) server and an accompanying UI Agent client.

Architecture 01

The Dedicated UI Agent Client

To seamlessly integrate natural language into the CMS interface, I developed a custom Gemini-powered Agent API that sits between the web UI and the MCP server. It manages multi-step task execution, conversational history, and long-polling events.

Crucially, it utilizes AgentContext: passing the user's current UI state (selected folders, open items) directly to the model. This allows the user to say "update this item" without explicitly citing complex URIs. Furthermore, the agent can emit UI Actions (e.g., requestNavigation), allowing the AI to physically drive the UI and navigate the user to the newly created content.

Architecture 02

High-Throughput Tool Orchestration

For complex bulk actions and reporting, standard LLM step-by-step execution is too slow and fragile. To resolve this, I developed the toolOrchestrator. The LLM generates a sandboxed JavaScript script utilizing loops and batch processing, which executes natively on the server. This allows massive reports to be compiled or hundreds of pages to be updated reliably.

Architecture 03

JIT Hydration & Future-Proofing

While modern LLMs can comfortably handle the 80+ complex API tools exposed by the CMS, enterprise environments rarely operate in isolation. To ensure the CMS MCP server can be seamlessly integrated into broader customer setups, I developed two specialized hydration patterns: Dynamic Registration (for architectural purity) and an Optimized Proxy (for broader client compatibility). These patterns utilize "discovery handshakes" to request specific tools just-in-time, optimizing the system prompt footprint while significantly reducing token costs and latency.
